# Naviory 隐私声明

**生效日期：2026-05-12**

Naviory 是一个本地优先的新标签页扩展。本声明说明它处理什么数据、如何处理、以及用户拥有哪些权利。

## 数据范围

Naviory 在本机浏览器的 IndexedDB 中保存以下数据：

- **分组**（名称、图标、颜色、排序）
- **链接**（标题、URL、描述、标签、所属分组、排序、创建时间）
- **应用设置**（主题、默认搜索引擎、链接打开方式）

数据库名：`naviory`。表：`groups`、`links`、`settings`。

## 数据传输

**Naviory 不向任何服务器发送数据。**

- 没有后端服务、没有 API 调用
- 没有遥测、没有埋点、没有错误上报
- 没有第三方分析（Google Analytics、Sentry 等）一概未集成
- 没有广告、没有跟踪像素

唯一对外的网络行为是：
- 用户主动从搜索框跳转到外部搜索引擎（Google / Bing / 百度等）
- 用户主动点击链接卡片跳转到外部网站

这两种行为与普通浏览器输入网址行为相同，由浏览器本身处理，扩展不参与数据传输。

## 不出售、不共享

我们不会：

- 出售用户数据
- 与第三方共享用户数据
- 将数据用于广告投放
- 将数据用于训练任何模型

## 浏览器权限

Manifest V3 申请的权限：

- `activeTab`：仅当用户主动点击扩展图标时，临时获取当前激活标签页的标题和 URL，用于"快速收藏"功能。**点击关闭后即失效。**

不申请的权限：

- 不申请 `<all_urls>` 或任何 `host_permissions`
- 不申请 `storage`（数据走 IndexedDB）
- 不申请 `tabs`、`history`、`bookmarks`、`cookies` 等敏感权限

## 用户权利

用户拥有完全控制权：

- **查看**：通过浏览器 DevTools → Application → IndexedDB 可直接查看所有原始数据
- **导出**：Options 页 "导出 JSON" 一键导出为人类可读的 JSON 文件
- **导入**：Options 页 "导入 JSON" 恢复或迁移数据
- **删除单项**：每条链接 / 分组都可单独删除（删除链接有 10 秒撤销）
- **清空全部**：Options 页 "清空全部数据" 要求输入 `DELETE` 二次确认后清空所有数据
- **卸载**：从浏览器卸载扩展即清除所有数据

## 数据丢失风险

由于数据仅在本机：

- 浏览器清理"网站数据 / Cookie"会丢失数据
- 卸载扩展会丢失数据
- 切换浏览器配置文件 / 更换电脑会丢失数据
- 浏览器存储被系统清理（极少数情况）也会丢失数据

请定期使用"导出 JSON"做备份。Options 页顶部黄色提示条提供"立即导出"快捷入口。

## 当前版本无云同步

本版本明确**不提供**：

- 云同步
- 账号系统
- 跨设备同步
- 任何形式的服务端存储

未来若添加上述功能，会在此声明明确更新，并在功能首次启用前征得用户同意。

# Naviory 手动测试清单

适用于：构建后加载 `dist/` 作为浏览器扩展进行人工验收。

每条用 `- [ ]` 形式列出；执行时按需在本地复制本文件并勾选。

---

## 0. 准备

- [ ] `pnpm install` 成功
- [ ] `pnpm typecheck` 0 错
- [ ] `pnpm lint` 0 错
- [ ] `pnpm test` 全部通过
- [ ] `pnpm build` 成功，`dist/` 生成

## 1. 扩展加载

### Chrome

- [ ] `chrome://extensions/` 开启开发者模式
- [ ] 加载已解压的扩展程序 → 选 `dist/`
- [ ] 扩展图标出现在工具栏
- [ ] 控制台无红色报错（service worker / popup / options / newtab 各页签均检查）

### Edge

- [ ] `edge://extensions/` 开启开发人员模式
- [ ] 加载解压缩的扩展 → 选 `dist/`
- [ ] 扩展图标出现在工具栏
- [ ] 控制台无红色报错

## 2. New Tab

- [ ] 打开新标签页 → 立即看到 Naviory 主界面
- [ ] 首次启动展示默认 7 个分组（工作 / 项目 / AI / 开发 / 文档 / 工具 / 临时）
- [ ] 顶部搜索框可获得焦点
- [ ] 右上角齿轮按钮存在
- [ ] 刷新页面后默认数据仍在
- [ ] 关闭浏览器再打开后数据仍在

## 3. Links CRUD

### 新增链接

- [ ] 分组内 "+ 添加" 弹出表单
- [ ] URL 不带协议时自动补 `https://`
- [ ] URL 非法时（如 `not a url`）弹错误提示，不允许保存
- [ ] 输入已有 URL 时显示 "已存在重复链接" 提示
- [ ] 提交后链接出现在分组中，刷新仍在

### 编辑链接

- [ ] hover 链接卡片显示编辑按钮
- [ ] 编辑后保存生效
- [ ] 把 URL 改成另一条已存在链接的 URL → 提示重复

### 删除链接

- [ ] 删除按钮触发 → 链接立即消失
- [ ] 同时右上 Toast 显示 `已删除链接` + `撤销` 按钮
- [ ] 10 秒内点 `撤销` → 链接恢复（id / 创建时间 / 排序保留）
- [ ] 不点撤销，10 秒后 Toast 自动消失，链接真删

## 4. Groups CRUD

### 新增分组

- [ ] 首页底部 "新增分组" 按钮可点
- [ ] 选择图标和颜色 → 保存后显示

### 编辑分组

- [ ] 修改分组名 / 图标 / 颜色生效

### 删除分组

- [ ] 空分组：触发 ConfirmDialog → 确认后删除
- [ ] 非空分组：弹窗显示 "下有 N 个链接，确定级联删除吗？" → 确认后分组及所有链接被删
- [ ] 取消按钮 / 关闭按钮工作正常

### 拖拽排序

- [ ] 长按链接卡片 → 可拖动，松手后顺序变化
- [ ] 刷新页面后顺序保留
- [ ] 跨设备同步时不应发生（本地版无此问题）

## 5. Search

- [ ] 在搜索框输入关键词 → 命中本地链接展示下拉
- [ ] 选中下拉项 → 跳转
- [ ] 未命中本地 + Enter → 走默认搜索引擎
- [ ] `g 关键词` Enter → Google
- [ ] `b 关键词` Enter → Bing
- [ ] `bd 关键词` Enter → 百度
- [ ] `gh 关键词` Enter → GitHub 搜索
- [ ] `npm 关键词` Enter → npm 搜索

## 6. Drag Sort

- [ ] 拖拽链接在分组内排序
- [ ] 拖拽分组（如支持）排序
- [ ] 排序结果在刷新后仍保留

## 7. Popup

- [ ] 工具栏图标点击弹出 Popup
- [ ] Popup 自动填入当前页 title 和 URL
- [ ] 分组下拉显示现有分组，可选择
- [ ] URL 重复时显示 "已存在" 提示
- [ ] 保存后 Toast 提示 + Popup 自动关闭
- [ ] 新增的链接在新标签页可见

## 8. Options

### 外观 / 搜索 / 交互

- [ ] 主题三态切换（跟随系统 / 浅色 / 深色）即时生效
- [ ] 默认搜索引擎切换后新标签页搜索默认引擎更新
- [ ] 链接打开方式切换后链接 click 行为符合配置

### 数据管理

- [ ] 导出 JSON → 下载 `naviory-backup-YYYY-MM-DD.json`
- [ ] 备份 JSON 包含 `appVersion` 字段（与 manifest 版本一致）
- [ ] 导入 JSON → 选择刚才导出的文件 → 二次确认后数据完全恢复
- [ ] 导入非法 JSON → 提示格式错误，DB 不变
- [ ] 扫描重复链接面板能正确列出重复项（造一对带/不带 utm_ 的同 URL 测试）
- [ ] 导入浏览器书签 (HTML) → 顶层文件夹成为分组，链接挂入对应分组
- [ ] 清空全部数据按钮 → 弹出 TypedConfirmDialog
- [ ] 不输入 / 输错 → 确认按钮 disabled
- [ ] 输入 `DELETE` → 按钮启用，确认后数据被清空并恢复默认 7 个分组

### 关于

- [ ] 显示当前版本号

## 9. Dark Mode

- [ ] 跟随系统：操作系统切换深色 → 扩展内自动切换
- [ ] 强制浅色：所有页面（newtab / popup / options）保持浅色
- [ ] 强制深色：所有页面保持深色
- [ ] BackupNotice、TypedConfirmDialog、DuplicatesPanel、Toast 在暗色下可读

## 10. 数据持久性

- [ ] 添加几条链接后刷新新标签页 → 数据仍在
- [ ] 关闭再打开浏览器 → 数据仍在
- [ ] 浏览器重启后 → 数据仍在
- [ ] 在 Chrome DevTools → Application → IndexedDB → naviory 中可看到 groups / links / settings 三张表

## 11. 上线前检查（Chrome Web Store 准备）

- [ ] manifest.json 中 `permissions` 仅包含 `activeTab`
- [ ] 无 `host_permissions`、无 `<all_urls>`
- [ ] 图标 16/48/128 都存在且可读
- [ ] 不存在远程脚本注入或动态 import 远程地址
- [ ] 隐私声明 `docs/privacy.md` 内容与实际行为一致
